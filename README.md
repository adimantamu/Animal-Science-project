# Mastitis Project — Dataset Construction (AFI + Rumination/Water + Events)

This repository contains a **complete data engineering pipeline** (implemented in a Jupyter Notebook) that consolidates multiple dairy farm data sources into one **daily, per-cow master dataset** for mastitis-related analysis.

The final artifact produced by the pipeline is:

- `Mastitis_Dataset_final.csv` — **master dataset** (one row per cow per day) enriched with:
  - milk yield
  - milk conductivity
  - pathogen metadata (bacteria / Gram / type)
  - rumination index (seconds)
  - water intake (liters)
  - farm management **events** as a cleaned list (e.g., `['OPEN']`, `['LAB', 'MAST']`)

---

## Project goal (what this pipeline accomplishes)

In real farm environments, milk and animal-health signals often come from **separate systems**:

- AFI (milk yield + conductivity + bacteriology labels for infected cows)
- “Before” milk dataset exports (Posthmus dairy daily milking logs)
- “After” milk dataset exports (daily CSV snapshots)
- Sensor streams (rumination + water intake logs at high frequency)
- Event logs (repro, mastitis flags, lab notes, moves, etc.)

This project’s core contribution is producing a **single, aligned daily timeline** for each cow and then **patching missing values** from multiple sources—while also attaching behavioral signals (rumination/water) and cleaned event annotations.

## Pipeline overview (step-by-step execution)

The logic below follows the implementation in `Animal_science_dataset_preparation.ipynb`.

### 1) Load AFI base dataset (labels + initial yield/conductivity)

The notebook first loads a labeled AFI “base” dataset and keeps only:

- `ID`, `DailyYield`, `Avg.Conductivity`, `Bacteria`, `Gram`, `Type`, `date`

This dataset contains the **pathogen context** for each cow (and sometimes yield/conductivity, with missing days).

### 2) Expand each cow into a continuous timeline (the “skeleton”)

For each `ID`, the notebook:

- finds the cow’s min/max date in AFI base data
- creates a **full daily range** from \(min\_date - 30\) days to \(max\_date + 30\) days
- merges the original rows onto this template
- forward-fills (specifically) `Bacteria`, `Gram`, and `Type` into newly created days:
  - **before** the original min date: filled using the min-date values
  - **after** the original max date: filled using the max-date values

This produces `afi_base_data_expanded`, which is the master “date skeleton” used for all later joins.

### 3) Load + normalize “before” milk dataset (Posthmus export)

The notebook loads a “before” dataset (daily milking logs) and extracts:

- `Animal_ID` → `ID`
- `Date` → `date`
- `Yield(gr)` → a yield candidate
- `Conductivity` → a conductivity candidate

This dataset is used to patch gaps where AFI base yield/conductivity is missing.

> Note: the notebook references an Excel file for this step. If it’s not present in your repo, you can still understand and reproduce the logic by supplying the same table structure.

### 4) Load + normalize “after” milk dataset (daily AFI snapshot CSVs)

From `AFI data/OneDrive_AFI_after data/*.csv`, the notebook concatenates all daily CSV snapshots and **extracts the date from each filename** (e.g. `TAMU - 03-30-2024 04-00.csv`).

From this merged table it selects:

- `Cow` → `ID`
- `Daily` → yield candidate
- `Avg.` → conductivity candidate
- `date` from filename

### 5) Patch yield + conductivity using priority fill

The merge step builds a single daily dataset `final_dataset` (7 columns initially):

1. Start with the **expanded skeleton**: `afi_base_data_expanded`
2. Join the “before” table on `(ID, date)`
3. Join the “after” table on `(ID, date)`
4. Convert relevant columns to numeric (coercing invalid strings to NaN)
5. Fill yield/conductivity using priority:

- `DailyYield` (from AFI base)  
  then `yield_post` (before export)  
  then `yield_after` (after snapshots)

- `Avg.Conductivity` (from AFI base)  
  then `cond_post` (before export)  
  then `cond_after` (after snapshots)

Finally it renames fields to:

- `yield`, `conductivity`, `bacteria`, `gram`, `type`

### 6) Join rumination + water intake (high-frequency → daily)

Sensor data lives in `rumination and water intake/` as many CSVs (per-cow).

The notebook:

- loads only the cow IDs relevant to the AFI cohort
- concatenates the matched files
- filters the high-frequency stream by keeping rows whose timestamp contains `'00:00'`
  - (this is a fast downsampling strategy used in the notebook)
- keeps only: `Timestamp`, `animal_id`, `rum_index_seconds`, `water_intake_liter`
- parses timestamp to date
- aggregates to one row per `(ID, date)`:
  - rumination: **mean**
  - water intake: **sum**

### 7) Clean + join events (raw → daily list)

From `events_23_24.CSV`, the notebook:

- parses `Date` into `date`
- groups by `(ID, date)`
- strips whitespace from event strings
- removes duplicates
- stores results as a consistent **sorted Python list** per day

### 8) Final export

The final join creates `master_df` with 10 columns and exports it to:

- `Mastitis_Dataset_final.csv`

## How to run this project (reproducible steps)

### Prerequisites

- Python 3.10+ recommended
- Jupyter Notebook or JupyterLab

Python packages used in the notebook:

- `pandas`
- `numpy`

### Setup (Windows / PowerShell)

Create a virtual environment and install dependencies:

```powershell
python -m venv .venv
.venv\\Scripts\\Activate.ps1
python -m pip install --upgrade pip
pip install pandas numpy jupyter
```

### Run the pipeline

Launch Jupyter and execute the notebook:

```powershell
jupyter lab
```

Open `Animal_science_dataset_preparation.ipynb` and run all cells from top to bottom.

When successful, you should see an export step that writes:

- `Mastitis_Dataset_final.csv`

## Notes / limitations (important to mention in a report)

- **Upstream source files**: the notebook references “before” and AFI base Excel files; they may not be present in this repo snapshot. The pipeline and joins are still fully documented in the notebook, and the repo already contains the final outputs.
- **Yield units**: some sources use `Yield(gr)` while others contain daily values that appear to be in different units/scales. The pipeline keeps a single `yield` column after patching, so downstream modeling should confirm consistent units (or normalize).
- **Rumination downsampling**: the notebook filters rumination/water logs using timestamps containing `'00:00'`. This is fast and workable, but it is not the same as aggregating the full day’s high-frequency signal unless the logs already store daily aggregates at midnight.
